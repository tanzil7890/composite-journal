Function are the basic infrastructure of C++ and it also often calles as modules.They are like miniature programs that can be combined to form larger programs. They allow complicated programs to be divided into manageable pieces.

A function groups a number of program statements into a unit and gives it a name. Thus, this units can then be invoked from other parts of the program. The most important reason to use functions is to aid in the conceptual organization of a program.

Function Declaration

The typical way of getting something done in a C++ program is to call a function to do it. Defining a function is the way you specify how an operation is to be done. A function cannot be called unless it has been previously declared.

A function declaration gives the name of the function, the type of the value returned (if any) by the function, and the number and types of the arguments that must be supplied in a call of the function.

The following is its format:

Function-type nameOfFunction ( parameter1, parameter2, ...) {
	.......
	statements //function body 
	.......
}

Where: ‚Ä¢ Function-type¬†: is the data type specifier of the data returned by the function. ‚Ä¢ nameOfFunction¬†: is the identifier by which it will be possible to call the function. ‚Ä¢ parameters¬†: (as many as needed): Each parameter consists of a data type specifier followed by an identifier, like any regular variable declaration (for example: int x) and which acts within the function as a regular local variable. They allow to pass arguments to the function when it is called. The different parameters are separated by commas. ‚Ä¢ statements¬†: is the function's body. It is a block of statements surrounded by braces { }.

Function Definition

Every function that is called in a program must be defined somewhere (once only). A function definition is a function declaration in which the body of the function is presented. 

void add(int, int) //Function declaration

void add(int a, int b) //Function definition
{
	int sum;  //function body
	sum = a+b; //function body
	cout<<"The sum of 'a' and 'b' is: "<<sum<<endl;  //function body
}

In the above program, void add(int, int) is a declaring function so that we can use it elsewhere and void add(int a, int b){ .... } is defining the purpose of the function. So it is known as Function Definition.

The type of the definition and all declarations for a function must specify the same type. The argument names, however, are not part of the type and need not be identical.

Parameter Passing

In this module we will see the ways of passing value to the parameter and how does the parameter react when function is called using Parameter Passing.

Types of Parameter

A variable in a function header is called an parameter. The arguments/parameters are used to pass information from the calling function to the called function. Parameters are also known as messenger.

  • Formal Parameter:
    • The function header contains return-type-specifier, function name and list of arguments with their declaration.
    • These arguments are called as formal arguments or dummy arguments. Formal arguments get their values from the actual arguments.

    Syntax:

    int biggest (int x, int y, int z)   //Formal Parameter
  • Actual Parameter
    • The function call statement contains name of the function and the list of arguments to be passed. These arguments or parameters are called as actual arguments.
    • The arguments can be constants, variables, or expressions

    Syntax:

    biggest (a, b, c);    //Actual Parameter

Anatomy of Parameter Passing

  • Formal parameters, n1 and n2
    • Type of data they expect
    • Names that will be used internal to the function to refer to the values
  • Actual paramters
    • Actual values input to the function code by the caller
